# ТЗ: навчання бота стилю каналу (tone of voice + візуальна айдентика)

Мета: бот вивчає, як ведеться канал (тексти, тон, структура, емодзі, рубрики,
кольори/айдентика зображень), і генерує НОВІ пости (текст + картинка одним
повідомленням) у тому ж стилі.

## 1. Способи «навчити» бота (input)

Власник дає матеріал одним із способів — бот будує/оновлює `StyleProfile`:

| # | Спосіб | Працює для | Що бот бере |
|---|--------|-----------|-------------|
| A | **Переслати 1–N постів** боту (`/learn` → форвард постів → `/done`) | будь-який канал (вкл. приватний) | текст + зображення кожного поста |
| B | **Дати посилання** на канал (`@username` або `t.me/...`) | **тільки публічний** (парс `t.me/s/<канал>`) | останні ~20–60 постів: текст + URL картинок |
| C | **MTProto-скрейп** (вхід під акаунтом, `style:scrape`) | будь-який, повна історія | весь архів текстів (+ можна картинки) |

Рекомендований мінімум для якісного стилю: **5–15 постів** із текстом і **3–5 зображень**.

## 2. Що бот аналізує (processing)

**Тексти → Claude (claude-opus-4-8):**
- tone of voice (тон, звертання, формальність, гумор)
- типова довжина, структура (абзаци, списки, заголовки)
- емодзі (які, як часто, де)
- рубрики/типи постів і їхні мовні маркери
- лексика/слогани, що повторюються
- патерн CTA (як кличуть до дії)

**Зображення → Claude Vision (бот «бачить» картинки):**
- **кольорогама** (3–6 hex-кольорів бренду)
- візуальний стиль (мінімалізм/яскраво/фото/градієнти/колажі)
- композиція, наявність тексту на зображенні, лого/водяний знак
- настрій/естетика

Результат зберігається у `StyleProfile`:
- `analysisJson` — текстовий профіль (ToV)
- `brandKitJson` — палітра + візуальна айдентика
- `referencesJson` — приклади постів (few-shot)
- `refImagesJson` — ключі/URL референс-зображень

## 3. Як бот це використовує (usage)

- **Текст** (GPT) генерується з урахуванням ToV-профілю + 2–3 приклади постів (few-shot).
- **Зображення** (GPT image) — промпт автоматично доповнюється brand kit:
  палітра (конкретні hex), стиль, «без тексту на картинці / з підписом — як у каналі».
- Пост виходить у тому ж тоні й з візуалом, що тримає айдентику.

## 4. Обовʼязкові зміни UX (за вимогою власника)

1. **Текст + картинка = ОДИН пост** (одне повідомлення `sendPhoto` з підписом),
   ніколи не два окремих. Claude тримає підпис ≤ 1024 символів.
2. **Бот показує зображення** у прев'ю (надсилає фото з підписом — WYSIWYG, як у каналі).
3. **Кнопка («написать нам») — опційна** і налаштовується **всередині бота**:
   `/settings` → увімк/вимк кнопку, змінити текст і посилання.
4. Перенавчання будь-коли: повторний `/learn` оновлює профіль.

## 5. Команди бота (підсумок)

- `/learn` — режим навчання: пересилай пости (текст+фото) → `/done`
- `/learnlink <@канал>` — спарсити публічний канал за посиланням
- `/settings` — кнопка CTA (вкл/викл, текст, посилання), стиль
- `/style` — показати поточний профіль стилю (ToV + палітра)
- `/new`, `/connect`, `/channels`, `/plan` — як зараз

## 6. Технічні обмеження (чесно)

- Веб-скрейп `t.me/s/` — лише **публічні** канали, віддає обмежену кількість
  останніх постів (пагінація `?before=`).
- Bot API не дає історію приватних каналів і не дає список підписників — для цього MTProto.
- Якість стилю прямо залежить від кількості/репрезентативності прикладів.
